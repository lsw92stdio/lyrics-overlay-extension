## 완료된 작업: 팝업 UI 설정/디자인 탭 분리 및 구조 최적화

### 1. 팝업 UI 너비 확장 및 디자인 탭 신설
- **문제**: 설정창 내 기능이 많아져 스크롤이 지나치게 길어지고, 다국어 환경(영어, 일본어 등)에서 탭 이름의 줄바꿈이 발생할 우려가 있었습니다.
- **해결**: `popup.css`에서 `body` 너비를 `380px`에서 `420px`로 확장하고, `popup.html`에 설정(Settings) 탭과 별개로 단색 팔레트 SVG 아이콘을 적용한 **디자인(Design)** 탭을 새로 추가했습니다.

### 2. 설정과 디자인 기능 분리 및 UI 재배치
- **해결**: 기존 설정 탭에 있던 '위치 지정', '애니메이션', '텍스트 외곽선', '고정 핀 색상' 등 모든 시각적 요소와 '사이트 설정 관리자' 영역을 새롭게 만든 디자인 탭으로 이관했습니다. 
- **추가 개선**: '영역 선택(btnSelectArea)' 및 '관리자 열기(btnOpenOptions)' 버튼의 패딩과 폰트 사이즈를 셀렉트 박스(`padding: 6px 10px; font-size: 12px;`)와 완벽하게 일치시켜 UI 높이를 통일했습니다.

### 3. 저장 및 초기화(Reset) 로직 분리 (popup.js)
- **문제**: 탭이 분리됨에 따라, 각 탭 하단의 고정 버튼(저장, 초기화)이 자신이 속한 탭의 설정만 다뤄야 했습니다.
- **해결**: `GENERAL_KEYS`와 `DESIGN_KEYS` 상수를 선언하여 각각의 설정 키를 논리적으로 분리했습니다. 
  - `btnSaveSettings` / `btnResetSettings`: 동작, 연동, 표시 언어 등 일반 설정만 덮어쓰기/초기화합니다.
  - `btnSaveDesign` / `btnResetDesign`: 시각적 디자인 설정만 덮어쓰기/초기화합니다. (사이트별 디자인 덮어쓰기가 설정되어 있으면 개별 사이트 데이터(`siteStates`)에 저장됨)

### 4. 데이터 내보내기/가져오기(백업) 이원화
- **해결**: 백업 및 복원 그룹에 `lbl_design_data` 항목을 추가하고, `btnExportDesign` / `btnImportDesign` 버튼 및 전용 `file input`을 구현했습니다.
  - **설정 데이터**: 일반 설정(`GENERAL_KEYS`)과 리모컨 위치 정보만 내보내기/가져오기
  - **디자인 데이터**: 디자인 설정(`DESIGN_KEYS`), 사이트별 데이터(`siteStates`), 가사창 절대 위치 정보만 내보내기/가져오기

## 완료된 작업: 사이트별 개별 설정 기능 최적화 및 각종 UI/위치 버그 수정

### 1. 팝업 UI 초기 설정 적용 버그 수정 (popup.js)
- **문제**: 사용자가 개별 디자인이 적용된 사이트에서 팝업창을 열었을 때, 셀렉트 박스(designTargetSelect)가 '현재 사이트'가 아닌 '기본 설정(Global Default)'으로 강제 지정되어, 실수로 기본 설정을 덮어쓰게 되는 치명적인 버그가 있었습니다.
- **해결**: loadSettingsUI(keepToggleState = false, isInitialLoad = false)로 인자를 확장하고, init() 함수 내에서 처음 loadSettingsUI(false, true)를 호출할 때 siteStates[hostname].styleOverrides가 존재하면 자동으로 designTargetSelect.value = 'site'가 되도록 분기 처리했습니다.

### 2. 백그라운드 탭 가사창 위치 초기화(리셋) 버그 수정 (content.js)
- **문제**: 설정창에서 디자인을 변경하고 저장하면 모든 탭에 UPDATE_STYLE 메시지가 전송되어 pplySettings()가 실행됩니다. 이때 백그라운드에 가려진 탭은 브라우저 렌더링 최적화 때문에 ox.getBoundingClientRect()가 0을 반환하여 가사창 좌표가 (0,0)으로 강제 초기화되는 버그가 발생했습니다.
- **해결**: pplySettings() 내부에서 화면의 현재 좌표를 즉석에서 계산하는 코드를 제거하고, 스토리지에 마지막으로 저장된 안전한 절대 좌표만 읽어와 복구하는 estoreSavedPosition()을 직접 호출하도록 구조를 변경했습니다.

### 3. 하단 고정 버튼 영역 레이아웃 이탈 수정 (popup.html / popup.css)
- **문제**: popup.html의 사이트 설정 관리자 UI 등을 묶는 과정에서 닫는 </div> 태그가 하나 누락되었습니다. 이로 인해 #tab-settings 하단에 플렉스(flex)로 풋터 고정(fixed)되어야 할 .settings-actions-fixed 버튼 영역(기본값 복원, 저장 버튼)이 스크롤 영역(.settings-scroll) 안으로 딸려 들어가 버리는 UI 버그가 발생했습니다.
- **해결**: 누락되었던 닫는 </div>를 정확한 위치에 추가하여 DOM 트리를 정상 복구하고 하단 버튼 고정을 되살렸습니다.

### 4. 사이트 설정 관리자(Options Page) 문구 간소화 (messages.json)
- **문제**: 팝업창의 '사이트 설정 관리자' 설명 텍스트가 불필요하게 길어 UI를 답답하게 만들었습니다.
- **해결**: 다국어 번역 파일(_locales/ko, en, ja)의 lbl_site_manager_desc 항목을 대폭 축약했습니다.
